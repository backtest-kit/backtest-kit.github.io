BacktestUtils

Utility class for backtest operations.

Provides simplified access to backtestCommandService.run() with logging. Exported as singleton instance for convenient usage.

Memoized function to get or create BacktestInstance for a symbol-strategy pair. Each symbol-strategy combination gets its own isolated instance.

Runs backtest in background without yielding results.

Consumes all backtest results internally without exposing them. Useful for running backtests for side effects only (callbacks, logging).

stop

stop: (symbol: string, strategyName: string) => Promise<void>

Stops the strategy from generating new signals.

Sets internal flag to prevent strategy from opening new signals. Current active signal (if any) will complete normally. Backtest will stop at the next safe point (idle state or after signal closes).

cancel

cancel: (symbol: string, strategyName: string, cancelId?: string) => Promise<void>

Cancels the scheduled signal without stopping the strategy.

Clears the scheduled signal (waiting for priceOpen activation). Does NOT affect active pending signals or strategy operation. Does NOT set stop flag - strategy can continue generating new signals.
